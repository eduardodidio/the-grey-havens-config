# F03-T03 — Codex mapper: degrade `reasoning` to raw per documented contract

**Wave:** 1
**Type:** backend
**Depends on:** F03-T01
**Status:** planned

## User Story

As a dashboard/error-detection consumer, I want a Codex `reasoning` item (which
has no Claude analogue) to normalize to `kind="raw"` with its category
preserved, so that the normalizer matches its own documented contract and never
mislabels reasoning as a tool.

## Objective

Corrigir `bin/didio-events-lib.py` para que o item Codex `reasoning` degrade
para `kind="raw"` (category `reasoning`), conforme o docstring do módulo,
mantendo os outros 12 testes de `bin/test_didio_events.py` verdes. **Não tocar**
o arquivo de teste.

## Dev Notes

Veja `_brief.md` e `F03-root-cause.md` (de F03-T01). Único arquivo de produção a
editar: `bin/didio-events-lib.py`. Não editar `bin/test_didio_events.py` (ele já
expressa o comportamento correto, linhas 88–93).

Contradição a resolver: o docstring (linhas 24–26 e 88–89) cita `reasoning` como
o exemplo canônico de "no Claude analogue → degrade to raw", mas a linha 101
lista `"reasoning"` em `_CODEX_TOOL_ITEM_TYPES`, fazendo o item virar
`kind="tool"`. O fallback de `item.completed` (linha 148) já produz
`{"kind":"raw","category":item_type,...}` — basta o item cair nele.

Fixture relevante (`tests/fixtures/codex-json.jsonl:1`):
```
{"type":"item.completed","item":{"type":"reasoning","text":"Thinking about the task."}}
```

`count_tool_errors` (linha 233) NÃO inclui `"reasoning"` nas categorias de erro,
então remover do tool-set não muda a contagem de erros do Codex (esperada = 1).

## Implementation details

Em `bin/didio-events-lib.py`, linha 101, remover `"reasoning"` do conjunto:
```python
# antes
_CODEX_TOOL_ITEM_TYPES = {"command_execution", "mcp_tool_call", "file_change", "web_search", "reasoning"}
# depois
_CODEX_TOOL_ITEM_TYPES = {"command_execution", "mcp_tool_call", "file_change", "web_search"}
```
Resultado: `item_type == "reasoning"` não casa nenhum ramo e cai no fallback da
linha 148 → `{"kind":"raw","category":"reasoning","is_error":False,...}`.

Opcional (clareza, não obrigatório): ajustar o comentário ao lado do conjunto
para reafirmar que `reasoning` degrada para raw. Não introduzir dependências.
