## User Story

As a framework maintainer, I want a single `tests/run.sh` that discovers and
runs every test, so that the project has a real Test command and later Waves
(and CI) can validate everything with one invocation.

## Objective

Create `tests/run.sh` aggregating all shell + python tests, and replace the
placeholder Build/Test/Run lines in `CLAUDE.md` with real commands.

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/05-foundations.md`.

- Existing tests to discover: `tests/F01-*.sh` (4 files), `tests/F02-*.sh`
  (added by later F02 tasks), `bin/test_didio_events.py`,
  `bin/test_didio_progress.py` (global), plus any `tests/test_*.py`.
- **Discovery via glob, not a hardcoded list** — new F02 test files must be

  (single-writer rule).
- Dependency-free: bash + `python3` only. `shellcheck` is NOT required here

- CLAUDE.md placeholders live at lines 18–20 (`Build:` / `Test:` / `Run:`).
- Style: print a per-file `PASS`/`FAIL`, a final summary
  (`N passed, M failed`), exit non-zero if any failed.

## Implementation details

- New file `tests/run.sh` (`#!/usr/bin/env bash`, `set -uo pipefail`):
  - `ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"`.
  - Run each `"$ROOT"/tests/F0*-*.sh` (skip `lib/` and `fixtures/`), capture
    exit code.
  - Run each Python test found via `"$ROOT"/bin/test_*.py` and
    `"$ROOT"/tests/test_*.py` with `python3 <file>` (or `-m pytest` if a
    `pytest` shim is detected — but default to plain `python3` execution,
    since `test_didio_events.py` is unittest/`__main__`-runnable).
  - Accumulate failures; print summary; `exit 1` on any failure.
- `CLAUDE.md`:
  - **Test:** `bash tests/run.sh`
  - **Build:** `bash tests/F02-shellcheck.sh` (lint; no compiled artifact) —
    or a no-op note if T02 not yet present; keep wording tolerant.
  - **Run:** `didio <subcommand>` (the CLI entrypoint; note it's the global
    install).

## Acceptance criteria

- [ ] `bash tests/run.sh` runs all F01 shell tests + `test_didio_events.py`
      and reports an accurate pass/fail summary.
- [ ] Exit code is 0 when all pass, non-zero when any fails (verified by a
      temporary failing stub during dev, then removed).
- [ ] New `tests/F02-*.sh` files are auto-discovered (no runner edit needed).
- [ ] `CLAUDE.md` Build/Test/Run no longer contain `echo 'configure ...'`.
- [ ] Runner uses only bash + python3 (no new dependency).

## Testing

- Framework: bash. Command: `bash tests/run.sh` (self-validating).
- Test files live in `tests/`. The runner IS the harness; validate it by
  observing it correctly reports a known-passing suite and a deliberately
  failing temp file (then delete the temp file).

## Test scenarios

- **Happy path:** all current tests pass → summary `X passed, 0 failed`, exit 0.
- **Edge case:** a test directory with no python tests still runs shell tests
  cleanly (no "file not found" noise from an empty glob — guard `nullglob`
  or existence-check).
- **Error scenario:** inject a temp `tests/F02-_fail.sh` that exits 1 → runner
  exits non-zero and names the failing file; remove the temp file after.
- **Boundary:** runner invoked from a different CWD still resolves `ROOT`
  correctly (uses `BASH_SOURCE`, not `pwd`).
