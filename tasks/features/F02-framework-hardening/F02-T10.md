## Testing

- Framework: bash (`grep`). New `tests/F02-docs-consistency.sh`, also run via
  `bash tests/run.sh`.
- Scope: extracts every `didio <token>` from `agents/**` + `.claude/commands/**`
  and checks each against the repo-script + documented-global-command allow-list.
- Edge/error: a `didio spawn-agent` reference resolves to the repo script; an
  injected `didio frobnicate` in a temp doc copy must fail the gate (temp removed).

## Test scenarios

- **Happy path:** current docs (post-edit) → consistency test passes.
- **Edge case:** a `didio spawn-agent` reference resolves to the repo script.
- **Error scenario:** inject `didio frobnicate` into a temp doc copy → test
  flags it; remove temp.
- **Boundary:** a global-only command (`run-wave`) passes because it's
  allow-listed + documented, not because it's a repo script.
