## User Story

As a maintainer, I want an automated check that no secrets or agent
transcripts are committed, so that JSONL logs (which may contain secrets) and
credential files can never leak into git.

## Objective

Audit `.gitignore` coverage and add `tests/F02-secrets-scan.sh` that fails if
any log/meta artifact is tracked or any obvious secret pattern appears outside
fixtures.

## Dev Notes

Veja `_brief/04-security-hardening.md` (D5) e CLAUDE.md "Guardrails".

- Current `.gitignore` already covers: `logs/agents/*.jsonl`,
  `logs/agents/*.meta.json`, `logs/agents/state.json`,
  `logs/agents/*.checkpoint.json`, `logs/agents/*.ckpt.at`,
  `logs/agents/meta-metrics.json`, `logs/agents/_pending/`, `logs/*.log`,
  `logs/*.sh`, `logs/session-budget.json`, `logs/session-paused.json*`,
  `__pycache__/`, `*.pyc`.
- CLAUDE.md forbids committing `.env`, `credentials.*`, `*.pem`, `*.key`,
  tokens. Verify `.gitignore` also excludes these (add if missing).
- Fixtures (`tests/fixtures/*.jsonl`) are SAFE sample data and must be
  **allow-listed** by the scanner (don't flag them).
- Dependency-free: `git` + `grep` only.

## Implementation details

- `.gitignore`: add (if absent) `*.pem`, `*.key`, `.env`, `.env.*`,
  `credentials.*`, `*.p12`, keeping existing entries. Do not exclude the
  template/example configs.
- `tests/F02-secrets-scan.sh`:
  - **Check A — no tracked artifacts:** `git ls-files` must NOT list any
    `logs/agents/*.jsonl|*.meta.json|*.checkpoint.json|*.ckpt.at` nor any
    `.env`/`*.pem`/`*.key`/`credentials.*`. Fail listing offenders.
  - **Check B — secret patterns:** grep tracked files (excluding
    `tests/fixtures/`, `docs/`, `_brief/`) for `BEGIN .* PRIVATE KEY`,
    `AKIA[0-9A-Z]{16}`, `xox[baprs]-`, `ghp_[0-9A-Za-z]{20,}`,
    `-----BEGIN`, generic `api[_-]?key\s*[:=]\s*['\"][^'\"]{16,}`. Fail on hit.
  - Exit non-zero with a clear report on any violation.

## Acceptance criteria

- [x] `.gitignore` excludes `*.pem`, `*.key`, `.env*`, `credentials.*` (and
      keeps all existing log/meta entries).
- [x] `bash tests/F02-secrets-scan.sh` passes on the current tree.
- [x] The scan fails (verified with a temp tracked dummy) if a fake
      `logs/agents/x.jsonl` or a fake private key is staged; temp removed after.
- [x] Sample fixtures under `tests/fixtures/` are not flagged.

## Testing

- Framework: bash (`git` + `grep`). New `tests/F02-secrets-scan.sh`, also run
  via `bash tests/run.sh`.
- Scope: Check A asserts no `logs/agents/*` artifact, `.env`, `*.pem`, `*.key`
  or `credentials.*` is tracked; Check B greps tracked files for secret patterns.
- Edge/error: a staged `.env`/`*.pem`/`logs/agents/*.jsonl` must fail with the
  offending path named; `tests/fixtures/` sample data must NOT be flagged.

## Test scenarios

- **Happy path:** clean tree → exit 0.
- **Edge case:** allow-listed fixture JSONL present → not flagged.
- **Error scenario:** a staged `.env` / `*.pem` / `logs/agents/*.jsonl` → fail
  with the offending path named.
- **Boundary:** a secret-looking string inside `tests/fixtures/` or `_brief/`
  is ignored (scoped excludes), but the same pattern in `bin/` fails.
