## User Story

As a framework operator, I want config reads/writes to pass data to Python via
`sys.argv`/env instead of f-stringing into `python3 -c`, so that a config path,
key, or value containing quotes/newlines/code cannot break or inject the
script.

## Objective

Refactor every `python3 -c "..."` block in `bin/didio-config-lib.sh` to take
its inputs from `sys.argv`/`os.environ` with a **quoted heredoc** body, while
preserving exact stdout and the existing type-coercion semantics (F01 tests
must stay green).

- **Vulnerable functions** (all interpolate shell vars into Python source):

- **Fix pattern already in the repo** — `bin/didio-spawn-agent.sh:158`:
  ```bash
  python3 - "$META_FILE" "$FINAL_STATUS" "$EXIT_CODE" "$PHRASE" <<'PY'
  import json, sys
  path, status = sys.argv[1], sys.argv[2]
  ...
  PY
  ```
  The quoted `<<'PY'` prevents shell expansion; values arrive via `sys.argv`.
- **Preserve observable contract exactly:**
  - `didio_read_config`: bool→`true`/`false`, dict/list→`json.dumps`,
    else→`str(v)`; missing→empty; missing config→empty (`return 0` before).
  - `didio_write_config`: `true`/`false`→bool, `raw.isdigit()`→int, else str;
    create from template/`{}` if absent; `indent=2` + trailing newline.
  - role/provider getters: empty when unset; `didio_provider_for_role`
    defaults `claude`; `didio_provider_bin` defaults to the provider name;
    `didio_max_parallel` honors `turbo` (0) else `max_parallel`.
  - Keep `2>/dev/null || true` / default-echo fallbacks identical.
- Do NOT add `set -e` (file is sourced — see T02 policy).
- **Single-writer:** this task is the ONLY Wave-1 writer of
  `bin/didio-config-lib.sh`. Axis B (T11) edits only `didio.config.json`.
