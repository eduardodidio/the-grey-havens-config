## User Story

As a maintainer, I want a shellcheck baseline over all scripts, so that lint
regressions are caught and the intentional `set -e` differences are recorded
rather than "fixed" by accident.

## Objective

Add `tests/F02-shellcheck.sh` that lints `bin/*.sh`, `drivers/*.sh`,
`tests/*.sh`, and document the deliberate `set -euo pipefail` policy.

## Dev Notes

Veja `_brief/04-security-hardening.md` (D6) e `_brief/05-foundations.md`.

- `shellcheck` is installed at `/opt/homebrew/bin/shellcheck` but must be
  treated as **optional** (no new hard dependency): if absent, print a skip
  note and `exit 0`.
- **Policy to document (do NOT change code):**
  - `bin/didio-config-lib.sh` is **sourced** → must NOT `set -e` (would alter
    the caller's shell). It currently sets nothing global — intentional.
  - `drivers/{claude,codex,echo}-driver.sh` use `set -uo pipefail` (not
    `-e`) so they can capture the CLI exit code via `EXIT_CODE=$?` and
    propagate it — intentional, required by the driver contract.
  - `bin/didio-spawn-agent.sh` uses `set -euo pipefail` — correct (top-level
    executable).
- Record accepted suppressions as inline `# shellcheck disable=SCxxxx` with a
  one-line justification (the repo already uses `SC1090` for dynamic
  `source`).

## Implementation details

- New `tests/F02-shellcheck.sh` (`set -uo pipefail`):
  - `command -v shellcheck` guard → skip+exit 0 if missing.
  - Lint the three globs; treat warnings as informational, **errors as fail**
    (use `--severity=error` for the gate to avoid churn, and a second
    informational pass at default severity printed but non-fatal). Document
    the chosen severity in the file header.
  - Exclude generated/third-party paths if any (none expected).
- Add a short "Shell script conventions" note to `drivers/DRIVER_CONTRACT.md`
  **only if T06 hasn't claimed that file** — to avoid a same-Wave collision,
  put the `set -e` policy note **in this task's test-file header comment**
  and in `_brief` is already covered; do not edit DRIVER_CONTRACT here.

## Acceptance criteria

- [ ] `bash tests/F02-shellcheck.sh` exits 0 on the current tree at
      `--severity=error` (fix or justify any error-level finding).
- [ ] When `shellcheck` is absent, the script skips cleanly (exit 0, clear note).
- [ ] The `set -euo` policy is documented in the script header (sourced lib vs.
      drivers vs. executables) so future lint won't blanket-add `set -e`.
- [ ] No production script behavior changed by this task.

## Testing

- Framework: bash. Primary command: `bash tests/F02-shellcheck.sh`; also
  exercised via `bash tests/run.sh` (auto-discovered).
- Scope: lints `bin/*.sh`, `drivers/*.sh`, `tests/*.sh` at `--severity=error`
  (gate) plus an informational default-severity pass.
- Edge/error: when `shellcheck` is absent it skips cleanly (exit 0); a temp
  script with an error-level issue must be flagged, then removed.

## Test scenarios

- **Happy path:** clean tree → exit 0, "0 errors".
- **Edge case:** `shellcheck` not on PATH → skip note, exit 0.
- **Error scenario:** a temp script with an error-level issue is flagged
  (validate detection, then remove the temp).
- **Boundary:** a script with only warnings (no errors) does not fail the gate.
