## User Story

As a maintainer, I want a simulation that proves parallel Wave spawns don't
clobber each other's logs and that agents receive only their contract env, so
that the isolation guarantees the framework promises are regression-protected.

## Objective

Add `tests/F02-sim-parallel.sh` verifying: (a) concurrent spawns produce
distinct, non-overwritten log + meta files; (b) `didio_max_parallel` /
`didio_recommend_parallel` behave per config; (c) the driver receives ONLY the
documented `DIDIO_*` contract vars (context isolation).

## Implementation details

- `tests/F02-sim-parallel.sh`:
  - **No-clobber:** spawn N=3 distinct (role,task) combos into the same temp
    project's `logs/agents/`; assert 3 distinct `.jsonl` + 3 `.meta.json`,
    none truncated/overwritten (each contains its own role).
  - **Parallel safety:** optionally run the 3 spawns concurrently (`&` + wait)
    and assert all 3 logs/metas exist and are valid (no interleaving in a
    single file).
  - **max_parallel/recommend:** source config-lib; assert
    `didio_max_parallel` == config value, turbo override â†’ 0;
    `didio_recommend_parallel sonnet` returns the sonnet tier string.
  - **Isolation:** export `LEAK_CANARY=should_not_appear` in the parent, spawn
    with an instrumented driver, assert the canary is absent from the agent's
    observed env (driver writes observed env vars to the log).
  - Cleanup via `trap`.

## Acceptance criteria

- [ ] 3 concurrent spawns â†’ 3 distinct, valid log + meta files, none clobbered.
- [ ] `didio_max_parallel` returns the configured value and 0 under turbo;
      `didio_recommend_parallel` returns the correct tier string for sonnet.
- [ ] A parent-shell var (`LEAK_CANARY`) does NOT appear in the agent's env
      (context isolation proven); only `DIDIO_*` + inherited PATH reach it.
- [ ] Uses only echo/instrumented stand-in drivers (no model spend);
      deterministic; passes under `bash tests/run.sh`.
