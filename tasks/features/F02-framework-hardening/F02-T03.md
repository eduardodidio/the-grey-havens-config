## User Story

As a test author, I want a reusable simulation harness around
`didio-spawn-agent.sh` + `echo-driver`, so that the Axis C suite can simulate
agent management deterministically with zero model spend.

## Objective

Create `tests/lib/sim-harness.sh` (sourceable helpers) + any `tests/fixtures/
F02-*` needed, so `F02-sim-*.sh` (Wave 2) can spin an isolated project, spawn
agents, and assert on the produced `.jsonl` + `.meta.json`.

## Dev Notes

Veja `_brief/03-simulation-tests.md` e `_brief/05-foundations.md`.

- Reuse the **F01 test idiom**: `tests/F01-spawn-dispatch.sh` already sets
  `DIDIO_HOME="$ROOT_DIR"`, calls `bin/didio-spawn-agent.sh`, and asserts on
  meta. Copy its `assert_eq` / `assert_contains` helpers into the harness so
  Wave-2 tests share them.
- spawn-agent contract (grounded): `didio-spawn-agent.sh <role> <feature>
  <task-file> [extra]`; it sources `$PROJECT_ROOT/bin/didio-config-lib.sh`
  (project-local first), resolves provider via `didio_provider_for_role`,
  runs `drivers/<provider>-driver.sh`, writes
  `logs/agents/<F>-<role>-<task>-<TS>.jsonl` + `.meta.json`.
- `echo-driver.sh` writes ONE NDJSON line echoing `DIDIO_ROLE/FEATURE/TASK_ID/
  MODEL/FALLBACK/EFFORT`, exits `${ECHO_DRIVER_EXIT:-0}`.
- **Isolation:** the harness must build a temp `PROJECT_ROOT` (e.g.
  `mktemp -d`) containing `bin/` (symlink or copy of `didio-spawn-agent.sh`
  + `didio-config-lib.sh`), `drivers/echo-driver.sh`, `agents/prompts/*.md`
  (at least the roles tested), and a generated `didio.config.json` whose
  tested roles set `"provider": "echo"`. Spawn with `PROJECT_ROOT` and
  `DIDIO_HOME` pointed at the temp dir; always `rm -rf` on exit via `trap`.

## Implementation details

- New `tests/lib/sim-harness.sh` exposing:
  - `sim_make_project <tmpdir>` — scaffold the isolated project (bin, drivers,
    prompts, config with echo provider for given roles).
  - `sim_spawn <tmpdir> <role> <feature> <task-file> [exit_code]` — set
    `ECHO_DRIVER_EXIT`, run spawn-agent, echo the produced log+meta paths.
  - `sim_meta_field <meta.json> <key>` — read a meta field via `python3`
    (argv, not f-string — set the secure example).
  - `assert_eq`, `assert_contains`, `assert_file_exists`, test counters.
- New `tests/fixtures/F02-task.md` — a minimal valid task file body.
- Harness is **sourced**, so it must NOT `set -e` (mirror T02 policy).

## Acceptance criteria

- [ ] Sourcing `tests/lib/sim-harness.sh` defines the helpers without side
      effects and without altering the caller's `set` flags.
- [ ] `sim_make_project` produces a self-contained temp project that
      `didio-spawn-agent.sh echo`-dispatches successfully.
- [ ] A smoke assertion inside the harness file's `if [[ "${BASH_SOURCE[0]}"
      == "${0}" ]]` guard runs a single echo spawn and verifies log+meta exist
      (so `tests/run.sh` exercises it directly too).
- [ ] Temp dirs are always cleaned (`trap ... EXIT`); no writes to the real
      `logs/agents/`.

## Testing

- Framework: bash. Self-smoke command: `bash tests/lib/sim-harness.sh`; also
  exercised via `bash tests/run.sh`.
- Scope: sourcing defines helpers with no side effects; `sim_make_project`
  scaffolds an isolated temp project and one echo spawn produces log + meta.
- Edge/error: missing role prompt -> spawn-agent exits 2 (surfaced, no hang);
  `ECHO_DRIVER_EXIT=1` -> meta `failed`; temp dirs always cleaned via `trap`.

## Test scenarios

- **Happy path:** scaffold + one echo spawn → log line + meta `completed`.
- **Edge case:** role prompt missing → spawn-agent exits 2 (harness surfaces
  it, doesn't hang).
- **Error scenario:** `ECHO_DRIVER_EXIT=1` → meta `failed` (proves the knob
  works for Wave 2).
- **Boundary:** two spawns in the same temp project yield two distinct log
  filenames (timestamp/task-id uniqueness — foundation for T09).
