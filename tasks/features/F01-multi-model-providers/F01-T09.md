# Task F01-T09 — compile-skills core + Claude target emitter

**Feature:** F01
**Wave:** 2
**Type:** backend (compiler)
**Depends on:** F01-T03, F01-T06
**Status:** done

## User Story

As a maintainer, I want `didio compile-skills` to read `skills/` and emit the
Claude artifacts, so that the neutral source becomes the single source of truth
for `.claude/commands`, `.claude/agents`, and `agents/prompts`.

## Objective

Build `bin/didio-compile-skills.sh` (the compiler engine + Claude emitter): parse
front-matter, strip non-Claude override blocks, inline Claude blocks, write
GENERATED outputs idempotently. (Codex emitter is T10.)

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/02-skill-compile.md` (sections "Compiler"
+ "Neutral didio skill format" + the output-path mapping). Read `skills/SPEC.md`
(T03) and `skills/MIGRATION.md` (T06).

- File: `${DIDIO_HOME:-~/.claude-didio-config}/bin/didio-compile-skills.sh`
  (+ a `python3` helper inline or `bin/didio-compile-skills.py` for robust
  front-matter/block parsing — python3 is already a framework dep).
- Output mapping (Claude): `kind: command` → `.claude/commands/<name>.md`;
  `kind: subagent` → `.claude/agents/<name>.md`; `kind: role-prompt` →
  `agents/prompts/<name>.md`. Only emit when `claude ∈ targets`.
- Block handling: for target `claude`, remove `<!-- didio:codex -->…<!-- /didio:codex -->`
  entirely; unwrap `<!-- didio:claude -->…<!-- /didio:claude -->` (drop markers,
  keep content); leave shared lines and the two sentinels untouched.
- **Idempotency + GENERATED header:** every output starts with a comment marking
  it generated from `skills/<name>.md` — DO NOT EDIT. Re-running with no source
  change MUST produce no diff (the header must be deterministic — no timestamps).
- `--dry-run` prints what would change without writing (mirror
  `tests/F14-sync-dry-run.sh` style). `--target claude` (default all in T12).
- **AC2 (Claude half) + AC for T06 invariant:** compiled Claude command/prompt
  files must equal today's originals modulo the GENERATED header. Provide a
  compare mode or document the expected diff.

## Implementation details

- Deterministic ordering (sort skills by name) so output is stable.
- Write to a temp file then `mv` into place (atomic, no partial files).
- Subagents only emit for Claude (Codex skip handled in T10).

## Testing

- Framework: bash + python smoke test against `skills/_example.md` (T03) and the
  migrated skills (T06).
- Command: `bash ${DIDIO_HOME}/tests/F01-compile-claude.sh`
- Test file: `${DIDIO_HOME}/tests/F01-compile-claude.sh`
- Convention: compile into a temp output root; assert files exist, blocks
  stripped, header present, sentinels preserved, second run = no diff.

## Test scenarios

- **Happy path:** `_example.md` compiles to `.claude/commands/_example.md` with
  codex block removed and claude block unwrapped.
- **Edge case:** a `targets: [codex]`-only skill produces NO Claude output.
- **Error scenario:** a skill with malformed front-matter → non-zero exit + clear
  message naming the file; no partial output written.
- **Boundary values:** idempotency — running twice yields byte-identical output
  (diff empty); sentinels `{{USE_SECOND_BRAIN}}`/`{{DIDIO_CHECKPOINT}}` survive.
