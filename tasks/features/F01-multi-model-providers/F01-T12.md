## User Story

As a user, I want `didio compile-skills` and `didio providers` subcommands, so
that I can regenerate skill artifacts and verify provider CLIs from one entry
point.

## Objective

Wire the new subcommands into `bin/didio` and add a `bin/didio-providers.sh`
helper; extend the `--help` heredoc.

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/03-cli-config-surface.md`. The dispatcher
is `${DIDIO_HOME:-~/.claude-didio-config}/bin/didio` ŌĆö a `case "$SUBCMD"` that
`exec`s `bin/didio-*.sh`.

- Add cases:
  - `compile-skills)  exec "$DIDIO_HOME/bin/didio-compile-skills.sh" "$@" ;;`
    (script from T09/T10; supports `--dry-run`, `--target claude|codex|all`).
  - `providers)       exec "$DIDIO_HOME/bin/didio-providers.sh" "$@" ;;`
- New file `${DIDIO_HOME}/bin/didio-providers.sh`:
  - `providers list` ŌåÆ print each provider from config `providers{}` + the
    `command -v <bin>` resolution (found/missing).
  - `providers validate` ŌåÆ for every provider referenced by any role in the
    active config, assert `command -v <bin>` succeeds; exit non-zero listing any
    missing binary. Used by preflight (T13).
  - Source `bin/didio-config-lib.sh` for `didio_provider_bin` and to enumerate
    roles/providers.
- Extend the `help|-h|--help` heredoc in `bin/didio` with both subcommands.
- Keep existing cases untouched; unknown subcommand behavior unchanged.

## Implementation details

- `didio-providers.sh`: `set -euo pipefail`, friendly output, exit codes: 0 all
  present, 1 one+ missing (validate). `list` always exits 0.
- Only validate providers actually used by the current config (don't force codex
  on Claude-only projects).

## Testing

- Framework: bash smoke test with stubbed/absent binaries on PATH.
- Command: `bash ${DIDIO_HOME}/tests/F01-cli-subcommands.sh`
- Test file: `${DIDIO_HOME}/tests/F01-cli-subcommands.sh`
- Convention: temp config + manipulated PATH; assert `providers validate` exit
  codes and `compile-skills --dry-run` dispatches.

## Test scenarios

- **Happy path:** `didio providers list` shows claude+codex with resolution;
  `didio compile-skills --dry-run` runs the compiler in dry mode (no writes).
- **Edge case:** Claude-only config ŌåÆ `providers validate` ignores absent
  `codex` and exits 0.
- **Error scenario:** a role set to `codex` but `codex` not on PATH ŌåÆ
  `providers validate` exits 1 naming `codex`.
- **Boundary values:** `didio --help` lists both new subcommands; unknown
  subcommand still exits 1 with the original message.
