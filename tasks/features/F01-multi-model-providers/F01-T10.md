# Task F01-T10 — compile-skills Codex target (~/.codex/prompts + AGENTS.md)

**Feature:** F01
**Wave:** 3
**Type:** backend (compiler)
**Depends on:** F01-T09
**Status:** done

## User Story

As a Codex user, I want `didio compile-skills` to also emit Codex artifacts, so
that the same neutral skills produce `~/.codex/prompts/*.md` and an aggregated
`AGENTS.md`.

## Objective

Add the Codex emitter to `bin/didio-compile-skills.sh`: command/role-prompt
skills → `~/.codex/prompts/<name>.md`; project instructions + role prompts →
aggregated `AGENTS.md`; subagents skipped with a documented note.

- **Path safety:** `~/.codex/prompts/` is in the user's HOME and may not exist —
  `mkdir -p`. Respect a `--out-codex <dir>` override so tests don't write to the
  real `~/.codex` (default to `${CODEX_HOME:-$HOME/.codex}/prompts`).
- Idempotency + GENERATED header rules from T09 apply identically.
- Sentinels: `{{USE_SECOND_BRAIN}}`/`{{DIDIO_CHECKPOINT}}` are still substituted
  at spawn time, so leave them in the Codex outputs too (spawn-agent runs for
  both providers).

## Implementation details

- `--target codex|all` selects emitters; `all` runs Claude (T09) + Codex.
- Aggregated `AGENTS.md` ordering deterministic (sorted); include a generated
  header and a table of contents of included roles.
- Skipped subagents: print `[compile-skills] skip codex: <name> (subagent)`.

## Testing

- Framework: bash + python smoke test, writing to a temp `--out-codex` dir.
- Command: `bash ${DIDIO_HOME}/tests/F01-compile-codex.sh`
- Test file: `${DIDIO_HOME}/tests/F01-compile-codex.sh`
- Convention: compile `_example.md` + a role prompt with `--out-codex <tmp>`;
  assert prompt file + `AGENTS.md` content, claude blocks stripped, idempotent.

## Test scenarios

- **Happy path (AC2):** `_example.md` → `<tmp>/_example.md` with claude block
  removed, codex block unwrapped; a role prompt is included in `AGENTS.md`.
- **Edge case:** subagent skill is skipped (no Codex file) with the documented
  log line.
- **Error scenario:** unwritable `--out-codex` → non-zero exit + clear message,
  no partial AGENTS.md.
- **Boundary values:** running `--target all` twice = no diff in either Claude or
  Codex outputs; sentinels preserved in Codex prompts.
