**Type:** infra / docs
**Depends on:** (none)

## User Story

As a framework maintainer, I want a documented driver contract and a `drivers/`
directory, so that Wave 1+ can implement Claude and Codex drivers in parallel
against a stable interface.

## Objective

Create `${DIDIO_HOME}/drivers/` and `DRIVER_CONTRACT.md` defining the exact env
var inputs, NDJSON output, and exit-code semantics every driver must honor.

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/01-execution-provider-abstraction.md`
(section "Driver contract").

- New dir: `${DIDIO_HOME:-~/.claude-didio-config}/drivers/`.
- The contract MUST match how `didio-spawn-agent.sh` will call drivers in T07.
  Fix these exported input vars now (drivers + spawn-agent must agree):
  `DIDIO_PROMPT`, `DIDIO_MODEL`, `DIDIO_FALLBACK`, `DIDIO_EFFORT`,
  `DIDIO_LOG_FILE`, `DIDIO_ROLE`, `DIDIO_FEATURE`, `DIDIO_TASK_ID`.
- Output: native streaming NDJSON appended to `$DIDIO_LOG_FILE` (driver performs
  the redirect, exactly like today's `> "$LOG_FILE" 2>&1`).
- Exit: driver exits with the underlying CLI's exit code.
- Resolution convention: `drivers/<provider>-driver.sh` (so adding providers
  later = zero spawn-agent edits).
- This is scaffolding only — no driver logic yet (T05/T08). Add a `.gitkeep` or
  the `DRIVER_CONTRACT.md` so the dir is tracked.

## Implementation details

- File: `${DIDIO_HOME}/drivers/DRIVER_CONTRACT.md` — document inputs/outputs/exit,
  the naming convention, and a minimal "echo driver" example used by tests.
- File: `${DIDIO_HOME}/drivers/echo-driver.sh` (test fixture) — a trivial driver
  that echoes a canned NDJSON line and exits 0; used to validate the contract and
  later the dispatch in T07 without real model spend. `chmod +x`.

## Testing

- Framework: bash smoke test.
- Command: `bash ${DIDIO_HOME}/tests/F01-driver-contract.sh`
- Test file: `${DIDIO_HOME}/tests/F01-driver-contract.sh`
- Convention: invoke `echo-driver.sh` with the contract env vars; assert it
  writes NDJSON to `$DIDIO_LOG_FILE` and returns 0.

## Test scenarios

- **Happy path:** echo-driver writes ≥1 valid JSON line to `$DIDIO_LOG_FILE`,
  exit 0.
- **Edge case:** empty `DIDIO_MODEL`/`DIDIO_FALLBACK` (Claude default path) does
  not error.
- **Error scenario:** a driver that `exit 7` propagates 7 to the caller.
- **Boundary values:** `DIDIO_PROMPT` containing quotes/newlines is preserved
  (no shell-injection / truncation).
