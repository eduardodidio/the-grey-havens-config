# Task F01-T14 — Wire compile-skills into sync-project

**Type:** infra (integration)
**Depends on:** F01-T09, F01-T10, F01-T12
**Status:** done

## User Story

As a downstream project, I want `didio sync-project` to deliver freshly compiled
skill artifacts, so that adopting projects get Claude (and Codex, when used)
skills generated from the neutral source.

## Objective

Invoke `didio compile-skills` from `bin/didio-sync-project.sh` so synced projects
receive compiled `.claude/commands` / `.claude/agents` / `agents/prompts` (and
Codex artifacts when applicable), keeping the existing idempotent/additive
guarantees.

## Dev Notes

Veja `_brief/00-overview.md`, `_brief/02-skill-compile.md` (section
"Source-checked-in vs generated") e `_brief/05-docs-guardrails.md`.

- File: `${DIDIO_HOME:-~/.claude-didio-config}/bin/didio-sync-project.sh`. It
  currently `sync_dir`s `.claude/agents`, `.claude/commands`, `agents/prompts`,
  `agents/workflows` (copy-if-missing, idempotent, additive; creates a rollback
  git tag; merges settings.json allow arrays).
- Decision (document in ADR T15): **source `skills/` checked in; outputs
  generated.** So sync should compile from `skills/` into the framework's own
  `.claude/*` + `agents/prompts/*` first (so the template dir is fresh), then run
  the existing `sync_dir` copy into the target. Do not break the
  never-delete/never-overwrite-beyond-placeholder contract.
- Codex outputs target the **user** `~/.codex/prompts` + `AGENTS.md`, which are
  global, not per-target-project — gate Codex emission behind "is any role using
  codex?" and document it (avoid surprising HOME writes during a routine sync).
- Respect `CLAUDE.md` guardrails: no `git add -A`, no force ops; sync already
  tags a rollback point — preserve that.

## Implementation details

- Add a compile step (call `bin/didio-compile-skills.sh --target claude` for the
  template/source refresh) early in sync, guarded so a compile failure aborts
  with a clear message before any copy. Keep `--dry-run` honored if sync is run
  in a dry mode.
- Keep ordering: compile → existing sync_dir copies → settings merge → CLAUDE.md
  append. Update the sync action log lines.

## Testing

- Framework: extend the existing sync dry-run smoke style.
- Command: `bash ${DIDIO_HOME}/tests/F01-sync-compile.sh`
- Test file: `${DIDIO_HOME}/tests/F01-sync-compile.sh`
- Convention: temp target project; run sync; assert compiled commands present and
  that re-running is idempotent (no diff, no deletions). Mirror
  `tests/F14-sync-dry-run.sh`.

## Test scenarios

- **Happy path:** sync into a fresh temp project → compiled `.claude/commands/*`
  present; rollback tag created (existing behavior intact).
- **Edge case:** re-sync → idempotent, no overwrites beyond placeholders, no
  deletions.
- **Error scenario:** compile failure (bad skill) → sync aborts before copying,
  clear message, target unchanged.
- **Boundary values:** Claude-only project → no writes to `~/.codex` / `AGENTS.md`.
