## User Story

As the execution layer, I want config-lib helpers that resolve a role's provider
and the provider-correct model id, so that spawn-agent can dispatch to the right
driver with a valid model namespace.

## Objective

Add `didio_provider_for_role`, `didio_provider_bin`, and provider-aware model
resolution to `bin/didio-config-lib.sh`, defaulting to `claude` when unset.

## Implementation details

- `didio_provider_for_role <role>` â†’ reads `models[.economy].<role>.provider`,
  default `claude`.
- `didio_provider_bin <provider>` â†’ reads `providers.<provider>.bin`, default =
  the provider name itself (`claude`â†’`claude`, `codex`â†’`codex`).
- `didio_provider_model_for_role <role>` â†’ returns the role's `model` as-is (the
  model id is already provider-namespaced by the author in config); this helper
  exists so callers don't assume Claude tiers. Keep `didio_model_for_role` as the
  back-compat alias.
- Add nothing that runs at source-time; functions only.
