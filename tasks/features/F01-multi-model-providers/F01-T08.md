# Task F01-T08 — drivers/codex-driver.sh (codex exec --json --yolo)

**Feature:** F01
**Wave:** 2
**Type:** backend (execution)
**Depends on:** F01-T02
**Status:** done

## User Story

As a Codex-assigned role, I want a driver that runs `codex exec` headless and
streams NDJSON to the log, so that my run is logged and observable like a Claude
run.

## Objective

Implement `drivers/codex-driver.sh` honoring the T02 contract, mapping the
contract env vars to the Codex CLI.

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/01-execution-provider-abstraction.md`
(equivalence map + "Driver contract"). Read `drivers/DRIVER_CONTRACT.md` (T02)
and use `drivers/claude-driver.sh` (T05) as the structural pattern.

- File: `${DIDIO_HOME:-~/.claude-didio-config}/drivers/codex-driver.sh`,
  `chmod +x`.
- Invocation (prompt via stdin, since prompts are large/multiline):
  ```bash
  set +e
  codex exec \
    --json \
    --yolo \
    ${DIDIO_MODEL:+--model "$DIDIO_MODEL"} \
    - <<<"$DIDIO_PROMPT" \
    > "$DIDIO_LOG_FILE" 2>&1
  EXIT_CODE=$?
  set -e
  exit $EXIT_CODE
  ```
  Mapping: `--json` ≈ Claude `--output-format stream-json --verbose`; `--yolo`
  ≈ `--dangerously-skip-permissions` (bypass approvals+sandbox); `--model` same.
- **Codex has no `--fallback-model`.** `DIDIO_FALLBACK` is accepted but unused —
  document this as a known gap (single-model; no automatic fallback). Same for
  `DIDIO_EFFORT` (model/config-dependent, not a flag).
- Do NOT pass `--ignore-user-config` (we want the user's `~/.codex/config.toml`).
- Inherit PATH only; the codex binary must be findable (preflight T13 verifies).

## Implementation details

- Mirror claude-driver's `set +e/-e` exit-capture structure and the

- Add a header comment documenting the fallback/effort gaps and the event-schema

## Testing

- Framework: bash smoke test with a **stubbed `codex`** on PATH.
- Command: `bash ${DIDIO_HOME}/tests/F01-codex-driver.sh`
- Test file: `${DIDIO_HOME}/tests/F01-codex-driver.sh`
- Convention: fake `codex` script in temp dir prepended to PATH, echoes canned
  `--json` NDJSON and a chosen exit code; assert flags + stdin prompt + log
  output. No real Codex spend.

## Test scenarios

- **Happy path:** stub `codex` is called as `codex exec --json --yolo --model X`
  with the prompt on stdin; NDJSON in `$DIDIO_LOG_FILE`; exit 0.
- **Edge case:** empty `DIDIO_MODEL` → no `--model` flag (uses Codex default).
- **Error scenario:** stub `codex` exits 2 → driver exits 2.
- **Boundary values:** multiline `DIDIO_PROMPT` with quotes is delivered intact
  via the heredoc-string (stub echoes it back for assertion).
