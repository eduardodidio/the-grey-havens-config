# Task F01-T03 — Neutral skill format spec + skills/ scaffold

**Feature:** F01
**Wave:** 0
**Type:** infra / docs
**Depends on:** (none)
**Status:** done

## User Story

As a skill author, I want one neutral skill format with provider-override blocks,
so that I write a skill once and the compiler can emit Claude and Codex variants.

## Objective

Create `${DIDIO_HOME}/skills/` and `skills/SPEC.md` defining front-matter,
sentinels, and the provider-override block syntax that T06/T09/T10 consume.

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/02-skill-compile.md` (sections "Neutral
didio skill format" and "Provider-override blocks").

- New dir: `${DIDIO_HOME:-~/.claude-didio-config}/skills/`.
- Front-matter keys: `name`, `description`, `kind` (`command|role-prompt|
  subagent`), `targets` (subset of `[claude, codex]`), optional `role-bindings`.
- Keep existing sentinels intact: `{{USE_SECOND_BRAIN}}` (substituted by
  `didio-spawn-agent.sh` from `second_brain.enabled`) and `{{DIDIO_CHECKPOINT}}`
  (see `agents/prompts/_checkpoint-block.md`). The compiler must NOT resolve
  these — they are resolved later at spawn time.
- Override block markers (HTML comments so they're inert in any markdown):
  `<!-- didio:claude -->...<!-- /didio:claude -->` and the `codex` equivalent.
  Shared lines live outside blocks.
- This task is the SPEC + one tiny **example skill** proving the format; the bulk
  migration is T06.

## Implementation details

- File: `${DIDIO_HOME}/skills/SPEC.md` — full grammar, a worked example, and the
  output-path mapping table (command→`.claude/commands` / `~/.codex/prompts`;
  role-prompt→`agents/prompts` / `AGENTS.md`; subagent→`.claude/agents` /
  skip-on-codex).
- File: `${DIDIO_HOME}/skills/_example.md` — minimal `kind: command`,
  `targets: [claude, codex]` skill with one override block, used by T09/T10 tests.

## Testing

- Framework: bash smoke test.
- Command: `bash ${DIDIO_HOME}/tests/F01-skill-spec.sh`
- Test file: `${DIDIO_HOME}/tests/F01-skill-spec.sh`
- Convention: parse `_example.md` front-matter with `python3`/`awk`; assert
  required keys present and override-block markers balanced.

## Test scenarios

- **Happy path:** `_example.md` front-matter has all required keys; both
  `didio:claude` and `didio:codex` blocks are opened and closed.
- **Edge case:** a skill with `targets: [claude]` only (no codex block) is valid.
- **Error scenario:** an unbalanced override block (open without close) is
  detected by the validator (negative fixture).
- **Boundary values:** sentinels `{{USE_SECOND_BRAIN}}` / `{{DIDIO_CHECKPOINT}}`
  in the body survive verbatim (not stripped by the format check).
