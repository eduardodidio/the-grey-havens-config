# Task F01-T06 — Migrate existing commands + prompts into skills/ neutral source

**Feature:** F01
**Wave:** 1
**Type:** infra (content migration)
**Depends on:** F01-T03
**Status:** done

## User Story

As a skill author, I want the existing Claude commands and role prompts seeded
into `skills/` in neutral format, so that the compiler can regenerate today's
Claude artifacts and also produce Codex variants.

## Objective

Port `.claude/commands/*.md` and `agents/prompts/*.md` into `skills/*.md` using
the T03 SPEC, adding front-matter and provider-override blocks only where the two
CLIs genuinely differ.

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/02-skill-compile.md` (sections "Today's
skill surface" and the output-path mapping). Read `skills/SPEC.md` from T03.

- Sources (framework repo): `${DIDIO_HOME:-~/.claude-didio-config}/.claude/
  commands/*.md` (`kind: command`), `${DIDIO_HOME}/agents/prompts/*.md`
  (`kind: role-prompt`, including the 10 roles + `_checkpoint-block.md`),
  `${DIDIO_HOME}/.claude/agents/*.md` (`kind: subagent`).
- For each, create `skills/<name>.md` with front-matter (`name`, `description`
  from the source, `kind`, `targets: [claude, codex]` for commands/role-prompts;
  subagents `targets: [claude]` only — no Codex analogue).
- **Do not rewrite content.** The neutral body == the current body. Wrap in
  `<!-- didio:claude -->/<!-- didio:codex -->` ONLY the lines that differ:
  references to `CLAUDE.md`↔`AGENTS.md`, `.claude/commands`↔`~/.codex/prompts`,
  and slash-command invocation phrasing. Keep `{{USE_SECOND_BRAIN}}` /
  `{{DIDIO_CHECKPOINT}}` sentinels verbatim.
- The acceptance bar: compiling these (T09) for Claude reproduces today's files
  modulo the GENERATED header. Keep that invariant in mind while porting.

## Implementation details

- One `skills/<name>.md` per existing command, role prompt, and subagent.
- Maintain a short `skills/MIGRATION.md` listing source→skill mappings so T09/T10
  and reviewers can verify completeness.
- Do not delete the originals yet (T09/T14 own the generated-vs-source cutover);
  this task is additive.

## Testing

- Framework: bash smoke test.
- Command: `bash ${DIDIO_HOME}/tests/F01-skill-migration.sh`
- Test file: `${DIDIO_HOME}/tests/F01-skill-migration.sh`
- Convention: assert one `skills/*.md` exists per source file; front-matter
  parses; override blocks balanced; sentinels preserved.

## Test scenarios

- **Happy path:** every `.claude/commands/*.md` and `agents/prompts/*.md` has a
  corresponding `skills/*.md` with valid front-matter.
- **Edge case:** subagent skills carry `targets: [claude]` (no codex target).
- **Error scenario:** a source file with no front-matter description still yields
  a valid skill (description derived from the H1 / filename), no crash.
- **Boundary values:** a role prompt containing both sentinels keeps them
  byte-identical after porting (grep count matches source).
