# Task F01-T11 — Per-provider JSONL/event normalization

**Feature:** F01
**Wave:** 3
**Type:** backend (observability)
**Depends on:** F01-T07, F01-T08
**Status:** done

## User Story

As a dashboard/observability user, I want Codex NDJSON normalized to a common
event shape, so that error-detection, rate-limit parsing, and the dashboard work
for Codex runs without breaking the Claude path.

## Objective

Introduce a per-provider event mapper (keyed off `meta.provider`) that translates
Claude stream-json and Codex `--json` events into one internal shape, and route
the JSONL consumers through it (Claude path = identity / unchanged).

## Dev Notes

Veja `_brief/00-overview.md` e `_brief/04-jsonl-normalization.md` (full
component). This is the AC4 task.

- `.meta.json` already carries `provider` after T07; consumers read it to pick a
  mapper.
- **Locate the live consumers** (the mission's `didio-jsonl-errors.py` /
  `didio-rate-limit-lib.sh` are target names — find the real ones): grep `bin/`
  for `tool_result`, `is_error`, `usage`, `rate`; the raw-JSONL renderer is the
  dashboard `dashboard/src/components/AgentRunDialog.tsx` with types in
  `dashboard/src/lib/types.ts` and parsing in `dashboard/src/lib/`. The meta
  aggregator (`bin/didio-progress.py`, `bin/didio-log-watcher-loop.py`) is
  schema-agnostic — no parse change, just surface `provider`.
- Define a **common internal event shape**, e.g.
  `{ kind: "assistant"|"tool"|"result"|"error"|"usage"|"rate_limit", raw, ... }`.
  `claude` mapper = thin pass preserving today's fields (MUST NOT change Claude
  behavior — golden test). `codex` mapper = translate Codex `--json` events.
- **Graceful degradation is allowed (AC4):** when a Codex event has no analogue
  (e.g. no rate-limit reset timestamp), degrade to raw / "n/a" instead of
  crashing — and document the gap in the ADR/README (coordinate with T15/T17).
- Keep the mapper in one place (a small `bin/didio-events-lib.*` or a function
  module) so both shell and dashboard can share semantics; if the dashboard
  needs a TS mirror, keep the mapping table identical and unit-test both.

## Implementation details

- New module: `${DIDIO_HOME}/bin/didio-events-lib.py` (mapper + provider switch)
  consumed by error/rate-limit logic; mirror minimal mapping in
  `dashboard/src/lib/` for the modal renderer with a matching unit test.
- Do NOT branch provider logic inline across many files — centralize in the
  mapper; callers ask the mapper for normalized events.

## Testing

- Framework: python `unittest` for the mapper + dashboard vitest for the TS
  mirror (the dashboard already uses vitest: `dashboard/src/lib/*.test.ts`).
- Command: `python3 ${DIDIO_HOME}/bin/test_didio_events.py` and
  `cd ${DIDIO_HOME}/dashboard && npm test`
- Test files: `${DIDIO_HOME}/bin/test_didio_events.py`,
  `${DIDIO_HOME}/tests/fixtures/{claude-stream.jsonl,codex-json.jsonl}`,
  dashboard `*.test.ts`.
- Convention: canned NDJSON fixtures for both providers; assert mapping to the
  common shape and Claude golden-equality.

## Test scenarios

- **Happy path:** a Codex `--json` error event maps to `kind:"error"`; a Claude
  `tool_result` error maps to the same `kind:"error"` (unified).
- **Edge case (AC3):** the Claude fixture maps to exactly today's internal fields
  (golden compare — no regression).
- **Error scenario:** a Codex event type with no analogue → degraded `kind:"raw"`
  / "n/a", consumer does not throw.
- **Boundary values:** empty log / truncated last NDJSON line is tolerated (no
  crash, partial line skipped).
