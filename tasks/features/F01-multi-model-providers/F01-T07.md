## User Story

As a role, I want spawn-agent to launch me through my configured provider's
driver, so that Codex-assigned roles run on `codex exec` while unset roles stay
on Claude, byte-for-byte.

## Objective

Replace the inline `claude -p` block in `didio-spawn-agent.sh` with driver
dispatch resolved by `didio_provider_for_role`, and record `provider` in
`.meta.json`.

## Test scenarios

- **Happy path (codex-ish):** role mapped to a stub provider → its driver
  executes, NDJSON in log, meta `provider` correct, exit propagated.
- **Edge case (AC3):** role with no provider → resolves `claude`, claude-driver
  invoked with the exact today flag-set (golden compare vs T05 stub capture).
- **Error scenario:** provider with no `<provider>-driver.sh` → exit 2 + clear
  message; meta marked `failed`.
- **Boundary values:** driver exit 1 → meta `status:"failed"`, `exit_code:1`,
  `finished_at` set.
